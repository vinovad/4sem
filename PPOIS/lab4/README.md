
# Система контроля качества выпечки (Bakery Quality Control System)
**Архитектура: Model-View-Controller (MVC) с пассивной View**

Данная программная система предназначена для автоматизации контроля качества на производстве выпечки. Она позволяет отслеживать путь продукта от начальной стадии до упаковки, проверять его на соответствие стандартам (ГОСТ, внутренние стандарты), выдавать сертификаты и анализировать обратную связь от клиентов. В рамках лабораторной работы №4 система была расширена графическим пользовательским интерфейсом с сохранением полной функциональности командной строки.

## Основные возможности (GUI + CLI)
*   **Управление продукцией:** Добавление новых продуктов, отслеживание их статуса и текущего этапа производства через интуитивный интерфейс.
*   **Контроль этапов производства:** Поэтапное производство продукта (замес, выпечка, охлаждение, упаковка) с обязательной сертификацией на каждом этапе для перехода к следующему.
*   **Управление стандартами:** Проверка продуктов по трем встроенным стандартам качества (ГОСТ, Стандарт пекарни, Органический стандарт) с наглядным отображением результатов.
*   **Сертификация:** Выдача сертификатов соответствия продукта стандарту на конкретном этапе производства с отображением срока действия.
*   **Улучшение продукта:** Автоматическое "улучшение" характеристик продукта до уровня, требуемого выбранным стандартом, с визуализацией изменений.
*   **Обратная связь:** Добавление отзывов и рейтингов к продуктам через удобную форму.
*   **Аналитика:** Расчет среднего рейтинга продукта на основе отзывов с цветовой индикацией и рекомендациями.
*   **Сохранение состояния:** Автоматическое и ручное сохранение/загрузка состояния системы в JSON-файл.

## Архитектура MVC (Model-View-Controller)

В лабораторной работе №4 была реализована архитектура **Model-View-Controller (MVC) с пассивной View (Passive View)**. 

### Схема взаимодействия компонентов:
```
[Пользователь] → [View (MainWindow)] → [Controller (MainController)] → [Model (QualityController)]
                      ↑                            ↓                           ↓
                      └────────────────────────────┴───────────────────────────┘
                         (Controller обновляет View и Model)
```

### Распределение ответственности:

| Компонент | Класс | Обязанности |
|-----------|-------|-------------|
| **Model** | `QualityController` | Бизнес-логика, данные, правила обработки |
| **View** | `MainWindow`, диалоги | Отображение информации, захват пользовательского ввода |
| **Controller** | `MainController` | Обработка действий пользователя, обновление Model и View |

## Документация классов (Domain) - Модель

### `Product`
**Файл:** `domain/product.py`

Класс предназначен для представления и управления информацией о продукте на всех этапах его жизненного цикла.

**Как работает:** Класс хранит все данные о продукте и предоставляет методы для их изменения. При создании продукт имеет начальные значения атрибутов, которые не соответствуют стандартам, и начинает с первого этапа производства.

*   **Атрибуты:**
    *   `product_id`: Уникальный идентификатор продукта.
    *   `name`: Название продукта.
    *   `attributes`: Словарь характеристик качества продукта `["defects", "texture", "smell", "taste"]`.
    *   `quality_checks`: Список результатов проверок качества.
    *   `reviews`: Список отзывов о продукте.
    *   `certificates`: Список выданных сертификатов.
    *   `status`: Статус продукта (production, quality_check, approved, rejected).
    *   `current_stage`: Индекс текущего этапа производства.
    *   `PRODUCTION_STAGES`: Список названий этапов производства `["dough", "baking", "cooling", "packaging"]`.
*   **Методы:**
    *   `get_current_stage()`: Возвращает название текущего этапа производства.
    *   `update_attribute(attribute, value)`: Обновляет значение указанного атрибута качества.
    *   `add_review(review)`: Добавляет отзыв к продукту.
    *   `__str__()`: Возвращает строковое представление продукта.

### `QualityStandard` (Абстрактный базовый класс)
**Файл:** `domain/quality_standard.py`

Базовый класс для всех стандартов качества.

**Как работает:** Каждый стандарт определяет конкретные требования к качеству продукта. При проверке соответствия система сравнивает атрибуты продукта с требованиями выбранного стандарта.

*   **Атрибуты:**
    *   `standard_name`: Название стандарта.
    *   `criteria`: Словарь требуемых значений для атрибутов продукта.

### `GOSTStandard`
**Файл:** `domain/quality_standard.py`

Класс, представляющий государственный стандарт (ГОСТ), наследуемый от `QualityStandard`.

*   **Атрибуты:**

    *   `criteria`: Требования: отсутствие дефектов `["none"]`, хорошая текстура, запах и вкус `["good"]`.

    *   `criteria`: Требования: отсутствие дефектов `["none"]`, отличная текстура, запах и вкус `["excellent"]`.

### `OrganicBakeryStandard`

### `ProductionStage` (Абстрактный базовый класс)
**Файл:** `domain/production_stages.py`

Базовый класс для этапов производства, определяющий интерфейс проверки.

**Как работает:** Каждый этап проверяет конкретный атрибут продукта. Чтобы перейти к следующему этапу, продукт должен соответствовать требованиям текущего этапа и иметь действительный сертификат.

*   **Атрибуты:**
    *   `stage_name`: Название этапа.
*   **Методы:**
    *   `check(product)`: Абстрактный метод для проверки продукта на конкретном этапе. Должен быть переопределен в подклассах.

### `DoughStage` (Этап замеса)
**Файл:** `domain/production_stages.py`

Класс, представляющий этап замеса теста. Наследуется от `ProductionStage`.

*   **Методы:**

### `BakingStage` (Этап выпечки)
**Файл:** `domain/production_stages.py`

Класс, представляющий этап выпечки. Наследуется от `ProductionStage`.

*   **Методы:**
    *   `check(product)`: Проверяет атрибут `texture`. Возвращает сообщение о прохождении этапа, если текстура хорошая `["good"]` или отличная `["excellent"]`.

### `CoolingStage` (Этап охлаждения)
**Файл:** `domain/production_stages.py`

Класс, представляющий этап охлаждения. Наследуется от `ProductionStage`.

*   **Методы:**
    *   `check(product)`: Проверяет атрибут `smell`. Возвращает сообщение о прохождении этапа, если запах хороший `["good"]` или отличный `["excellent"]`.

### `PackagingStage` (Этап упаковки)
**Файл:** `domain/production_stages.py`

Класс, представляющий этап упаковки. Наследуется от `ProductionStage`.

*   **Методы:**
    *   `check(product)`: Проверяет атрибут `taste`. Возвращает сообщение о прохождении этапа, если вкус хороший `["good"]` или отличный `["excellent"]`.

### `Certificate`
**Файл:** `domain/certificate.py`

Класс предназначен для представления сертификата качества, выданного продукту на определенном этапе.

**Как работает:** При выдаче сертификата устанавливается срок действия (1 год). Система проверяет действительность сертификата перед разрешением перехода к следующему этапу производства.

*   **Атрибуты:**
    *   `certificate_id`: Уникальный идентификатор сертификата.
    *   `product_id`: Идентификатор продукта, которому выдан сертификат.
    *   `stage`: Этап производства, на котором выдан сертификат.
    *   `standard`: Стандарт качества, по которому проведена сертификация.
    *   `issue_date`: Дата выдачи сертификата.
    *   `expiration_date`: Дата истечения срока действия (через год).
*   **Методы:**
    *   `is_valid()`: Проверяет, действителен ли сертификат на текущую дату.
    *   `__str__()`: Возвращает строковое представление сертификата.

### `Review`
**Файл:** `domain/review.py`

Класс предназначен для представления отзыва о продукте.

**Как работает:** Класс хранит информацию об отзыве и проверяет корректность рейтинга (от 1 до 5). Отзывы используются для анализа качества продукта.

*   **Атрибуты:**
    *   `product_id`: Идентификатор продукта, к которому относится отзыв.
    *   `author`: Имя автора отзыва.
    *   `comment`: Текст отзыва.
    *   `rating`: Рейтинг продукта (целое число от 1 до 5).
    *   `date`: Дата и время написания отзыва.
*   **Методы:**
    *   `__str__()`: Возвращает строковое представление отзыва.

### `QualityController`
**Файл:** `domain/quality_controller.py`

Главный компонент модели (Model), управляющий продуктами, этапами производства и стандартами. Реализует основную бизнес-логику приложения.

**Как работает:** Контроллер качества является центральным элементом модели, связывающим все остальные классы предметной области. Он реализует бизнес-логику приложения и обеспечивает взаимодействие между различными компонентами системы.

*   **Атрибуты:**
    *   `products`: Список всех продуктов в системе.
    *   `quality_standards`: Словарь доступных стандартов качества.
    *   `production_stages`: Список экземпляров этапов производства.
*   **Методы:**
    *   `get_standard(standard_name)`: Возвращает объект стандарта по его имени.
    *   `add_product(name)`: Создает и добавляет новый продукт в систему.
    *   `show_products()`: Возвращает список кортежей (ID, имя) для всех продуктов.
    *   `get_product(product_id)`: Возвращает объект продукта по его ID.
    *   `has_certificate_for_current_stage(product_id)`: Проверяет наличие действующего сертификата для текущего этапа продукта.
    *   `advance_stage(product_id)`: Переводит продукт на следующий этап производства.
    *   `is_production_complete(product_id)`: Проверяет, завершено ли производство продукта.
    *   `add_quality_check(product_id, stage, result)`: Добавляет запись о результате проверки качества.
    *   `add_certificate(product_id, certificate)`: Добавляет сертификат продукту.
    *   `check_compliance(product_id, standard)`: Проверяет, соответствует ли продукт заданному стандарту качества.
    *   `certify_product(product_id, standard)`: Выдает сертификат продукту.
    *   `improve_product(product_id, standard)`: Улучшает атрибуты продукта до требований стандарта.
    *   `run_production(product_id)`: Запускает процесс производства.
    *   `add_review(product_id, author, comment, rating)`: Создает и добавляет новый отзыв к продукту.
    *   `analyze_reviews(product_id)`: Анализирует отзывы о продукте.
    *   `save_state(filename)`: Сохраняет состояние системы в JSON-файл.
    *   `load_state(filename)`: Загружает состояние системы из JSON-файла.

## Документация классов (Controller)

### `MainController`
**Файл:** `gui/controllers/main_controller.py`

Класс-контроллер, обрабатывающий действия пользователя и координирующий работу Model и View.

**Как работает:** Контроллер получает события от View (нажатия кнопок, выбор элементов), вызывает соответствующие методы Model, обрабатывает исключения и обновляет View. В отличие от MVP, View не имеет прямой ссылки на Controller - Controller сам управляет View.

*   **Атрибуты:**
    *   `model`: Ссылка на экземпляр `QualityController` (Модель).
    *   `view`: Ссылка на экземпляр `MainWindow` (Представление), устанавливается через `set_view()`.
*   **Методы:**
    *   `set_view(view)`: Устанавливает ссылку на представление (пассивная View).
    *   `get_products()`: Возвращает список всех продуктов из модели.
    *   `add_product(name)`: Добавляет новый продукт через модель и обновляет представление.
    *   `get_product(product_id)`: Получает продукт по ID с обработкой ошибок.
    *   `get_standards()`: Возвращает список доступных стандартов для отображения.
    *   `check_compliance(product_id, standard_name)`: Проверяет соответствие стандарту с обработкой результата.
    *   `improve_product(product_id, standard_name)`: Улучшает продукт до соответствия стандарту.
    *   `certify_product(product_id, standard_name)`: Выдает сертификат с обработкой исключений.
    *   `advance_stage(product_id)`: Переводит на следующий этап с проверкой сертификата.
    *   `get_production_status(product_id)`: Возвращает структурированную информацию о статусе продукта.
    *   `add_review(product_id, author, comment, rating)`: Добавляет отзыв с валидацией.
    *   `analyze_reviews(product_id)`: Анализирует отзывы и возвращает структурированный результат.
    *   `save_state(filename)`: Сохраняет состояние с обработкой ошибок.
    *   `load_state(filename)`: Загружает состояние с обработкой ошибок.

## Документация классов (View)

### `MainWindow`
**Файл:** `gui/views/main_window.py`

Главное окно приложения, реализующее представление (View) в архитектуре MVC.

**Как работает:** Окно содержит все элементы управления и отображения, но не содержит бизнес-логики. Все действия пользователя передаются в `MainController` (Controller), который определяет, что и как отображать. View пассивна - она только отображает то, что говорит ей Controller.

*   **Компоненты интерфейса:**
    *   **Список продуктов:** Таблица `Treeview` с колонками: ID, Название, Этап, Сертификаты, Отзывы.
    *   **Панель инструментов:** Кнопки для основных операций (Добавить, Обновить, Сохранить, Загрузить).

    *   **Панель действий:** Кнопки для операций с выбранным продуктом (Проверить, Улучшить, Сертификат, Следующий этап, Отзыв, Анализ).
    *   **Строка состояния:** Отображение текущего статуса системы.
    *   **Меню приложения:** Файл, Продукт, Справка.
*   **Методы:**
    *   `refresh_product_list()`: Обновляет список продуктов в таблице (вызывается Controller-ом).
    *   `on_product_select(event)`: Обработчик выбора продукта из списка (передает управление Controller-у).
    *   `display_product_info(info)`: Отображает детальную информацию о продукте (вызывается Controller-ом).
    *   `get_selected_product_id()`: Возвращает ID выбранного продукта.
    *   `add_product()`: Вызывает соответствующий метод Controller-а.
    *   `check_quality()`: Вызывает соответствующий метод Controller-а.
    *   `certify_product()`: Вызывает соответствующий метод Controller-а.
    *   `add_review()`: Вызывает соответствующий метод Controller-а.
    *   `analyze_reviews()`: Вызывает соответствующий метод Controller-а.
    *   `show_info(message)`, `show_error(message)`: Методы для отображения сообщений пользователю (вызываются Controller-ом).

## Точки входа в приложение

### CLI версия
**Файл:** `cli/main.py`

```bash
# Запуск консольной версии
python main.py
```

Главная функция, реализующая цикл интерфейса командной строки (CLI). Она создает экземпляр `QualityController` и обрабатывает пользовательский ввод, вызывая соответствующие методы контроллера. Это классическая реализация MVC для консоли, где роль View выполняет консольный вывод, а Controller обрабатывает ввод пользователя.

**Структура меню:**
```
Меню:
1. Добавить продукт
2. Список продуктов
3. Запустить производство (перейти к следующему этапу)
4. Проверить продукт по стандарту качества
5. Внести улучшения в продукт
6. Выдать сертификат продукту
7. Добавить отзыв
8. Анализировать отзывы
9. Сохранить состояние системы
10. Загрузить состояние системы
11. Выход
```

### GUI версия

# Запуск графической версии

## Логика работы системы (GUI)

### 1. Запуск приложения
- Система пытается автоматически загрузить состояние из `system_state.json`
- Главное окно отображает список загруженных продуктов (если есть)
- Controller обновляет View через `refresh_product_list()`
- Интерфейс готов к работе

### 2. Добавление продукта
- Пользователь нажимает кнопку "Добавить продукт"
- View открывает диалог `ProductDialog`
- После ввода названия и нажатия "Добавить", View вызывает `controller.add_product(name)`
- Controller:
  - Вызывает `model.add_product(name)`
  - Обновляет View через `view.refresh_product_list()` и `view.show_info()`
- Список продуктов автоматически обновляется

### 3. Проверка по стандарту
- Пользователь выбирает продукт в списке
- Нажимает кнопку "Проверить качество"
- View открывает диалог `QualityCheckDialog`
- Пользователь выбирает стандарт и нажимает "Проверить"
- View вызывает `controller.check_compliance(product_id, standard)`
- Controller:
  - Вызывает `model.check_compliance()`
  - Обрабатывает результат/исключения
  - Возвращает результат в View для отображения
- Диалог отображает результат с цветовой индикацией

### 4. Улучшение продукта
- Пользователь выбирает продукт и стандарт
- View вызывает `controller.improve_product()`
- Controller проверяет соответствие через модель
- Если продукт не соответствует, модель автоматически улучшает атрибуты
- Controller обновляет View с сообщением о результатах

### 5. Выдача сертификата
- Пользователь открывает диалог сертификации
- View отображает существующие сертификаты (данные от Controller-а)
- Пользователь выбирает стандарт и нажимает "Выдать сертификат"
- View вызывает `controller.certify_product()`
- Controller:
  - Проверяет соответствие через модель
  - Проверяет отсутствие действующего сертификата
  - Вызывает `model.certify_product()`
  - Возвращает результат в View
- View отображает результат с цветовой индикацией

### 6. Производственный цикл
- Пользователь нажимает "Следующий этап"
- View вызывает `controller.advance_stage()`
- Controller:
  - Проверяет наличие сертификата через `model.has_certificate_for_current_stage()`
  - Вызывает `model.advance_stage()` при наличии сертификата
  - Обновляет View с результатом
- При успехе продукт переходит на следующий этап, список обновляется

### 7. Работа с отзывами
- **Добавление:** View открывает диалог, собирает данные, вызывает `controller.add_review()`. Controller валидирует данные и вызывает модель.
- **Анализ:** View открывает диалог, Controller получает данные анализа через `model.analyze_reviews()` и передает их в View для отображения с цветовой индикацией.

### 8. Сохранение и загрузка
- **Автоматическое:** При выходе из программы Controller вызывает `model.save_state()`
- **Ручное:** View вызывает методы Controller-а для сохранения/загрузки
- Controller обрабатывает ошибки и обновляет View с результатом

## Сравнение CLI и GUI версий

| Характеристика | CLI версия | GUI версия |
|----------------|------------|------------|
| **View** | Консольный ввод/вывод | Графическое окно с элементами управления |
| **Controller** | `main()` функция | `MainController` класс |
| **Model** | `QualityController` | `QualityController` (одинаковый!) |
| **Взаимодействие** | Цикл while с input() | Событийно-ориентированное |
| **Обратная связь** | Текстовые сообщения | Цветовая индикация, диалоги |
| **Выбор продукта** | Ввод ID | Клик по строке в таблице |
